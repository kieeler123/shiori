🇯🇵 日本語
Q1. サポートページの設計をどのように改善しましたか？

A:
SupportLayout を導入し、ネストルーティングを活用しました。
これにより、共通UIを一元管理し、各ページはコンテンツ責務に集中できるようにしました。

Q2. なぜ Layout 分離が必要だったのですか？

A:
各ページが独自にレイアウトを持つと重複が発生し、保守性が下がります。
レイアウトを共通化することで、UIの一貫性と拡張性を確保しました。

Q3. デザインと構造の関係をどう考えますか？

A:
デザインは見た目だけではなく、構造と密接に関連しています。
今回の改善はUI統一とルーティング構造整理を同時に行いました。

🇺🇸 English
Q1. How did you improve the support page structure?

A:
I introduced a dedicated SupportLayout and implemented nested routing.
This centralized common UI and allowed each page to focus on content responsibility.

Q2. Why was layout separation necessary?

A:
When each page handles its own layout, duplication and inconsistency occur.
Separating layout improves maintainability and scalability.

Q3. How do you see the relationship between design and structure?

A:
Design is not just visual styling.
It must align with structural clarity and component responsibility.

🇰🇷 한국어
Q1. 고객센터 페이지 구조를 어떻게 개선했습니까?

A:
SupportLayout을 도입하고 중첩 라우팅을 적용했습니다.
공통 UI를 중앙화하고 각 페이지는 컨텐츠 책임에 집중하도록 분리했습니다.

Q2. 왜 레이아웃 분리가 필요했습니까?

A:
페이지별로 레이아웃을 관리하면 중복과 유지보수 비용이 증가합니다.
공통화함으로써 일관성과 확장성을 확보했습니다.

Q3. 디자인과 구조의 관계를 어떻게 보십니까?

A:
디자인은 단순한 시각 요소가 아니라 구조 설계와 연결됩니다.
이번 작업은 UI 통일과 라우팅 구조 개선을 동시에 수행한 사례입니다.
