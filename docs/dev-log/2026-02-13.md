📌 Theme 적용 충돌 버그 기록
1️⃣ 문제 상황

ThemeProvider에서 테마 상태 관리 중

Layout에서도 initThemeName() 호출

select 변경 시 localStorage만 변경되고 즉시 반영 안 됨

새로고침하면 정상 반영됨

일부 컴포넌트는 적용되고 일부는 안 바뀜

에러 메시지는 없었음.

2️⃣ 증상

data-theme는 바뀌는 것처럼 보임

CSS 변수 일부는 갱신되지 않음

새로고침 후에는 정상 동작

HMR 환경에서 즉시 반영되지 않음

3️⃣ 원인 분석
핵심 원인: DOM 수정 주체가 여러 개였음

다음이 동시에 document를 조작하고 있었음:

initThemeName() (Layout)

setThemeName() (직접 호출)

ThemeProvider 내부 useEffect

localStorage 변경

즉,

Theme 상태의 “주인”이 여러 명이었음


React 구조에서 DOM은 결과물이어야 하는데
여러 곳에서 직접 DOM을 수정하는 구조였음.

4️⃣ 근본 원인 정리

React 구조는 이렇게 되어야 함:

Single Source of Truth (ThemeProvider)
        ↓
state 변경
        ↓
useEffect
        ↓
document.documentElement.dataset.theme 반영


하지만 실제 구조는:

state → DOM 직접 수정
localStorage → DOM 직접 수정
Layout → DOM 직접 수정
Provider → DOM 직접 수정


책임이 분산되어 충돌 발생.

5️⃣ 해결 방법

DOM 수정 책임을 ThemeProvider 하나로 통합

initThemeName() 제거

select에서는 setThemeName만 호출

localStorage 저장은 Provider 내부에서만 처리

결과:

상태 → effect → DOM 반영 구조로 정리됨

새로고침 없이 정상 반영

테마 구조 단일 책임 원칙 확보

6️⃣ 배운 점

DOM은 직접 만지지 않는다. 상태로 제어한다.

전역 상태는 반드시 단일 책임 구조로 만든다.

에러가 없다고 정상 동작은 아니다.

HMR과 실제 렌더 타이밍은 다를 수 있다.

"새로고침하면 된다"는 건 구조 문제가 있다는 신호다.

7️⃣ 한 줄 교훈

상태의 주인을 정하지 않으면 버그는 반드시 생긴다.
